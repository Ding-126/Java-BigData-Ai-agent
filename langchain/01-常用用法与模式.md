# LangChain 常用用法与模式（Python）

本文补充 `langchain/README.md` 的细节：把“能落地的常用模式”按场景整理成可直接复用的代码片段（偏工程实践）。

## 1. 最小调用：ChatModel + 消息

```python
from langchain_core.messages import SystemMessage, HumanMessage
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.2)
resp = llm.invoke(
    [
        SystemMessage(content="你是一个严谨的技术助手。"),
        HumanMessage(content="用 3 句话解释什么是 LangChain。"),
    ]
)
print(resp.content)
```

要点：

- **把 system prompt 当作“行为约束”**，把 user prompt 当作“输入”
- 尽量让输出“短、结构化、可解析”

---

## 2. 推荐写法：LCEL 管道（prompt | llm | parser）

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你是资深后端工程师，回答要点化、可执行。"),
        ("human", "给我一份 {topic} 的快速上手清单（不超过 {n} 条）。"),
    ]
)

chain = prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0.2) | StrOutputParser()
print(chain.invoke({"topic": "LangChain", "n": 8}))
```

要点：

- **Prompt 与模型解耦**：后续替换模型/温度/路由策略更容易
- **Parser 统一出口**：减少业务代码里对“模型输出长什么样”的依赖

---

## 3. 结构化输出（建议优先使用“强约束 + 校验”）

思路：

- Prompt 明确要求 JSON schema / 字段含义
- 模型输出后做 JSON 解析 + 校验（失败则重试或降级）

示例（思路演示，具体实现随模型/provider 能力会有差异）：

```python
import json
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你只能输出 JSON，不要输出多余文本。"),
        (
            "human",
            "把下面的文本提取为 JSON：{text}\n"
            "字段：language(string), keywords(array[string], <=5)",
        ),
    ]
)

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
msg = (prompt | llm).invoke({"text": "LangChain 是一个用于构建 LLM 应用的框架。"})

data = json.loads(msg.content)
assert isinstance(data.get("language"), str)
assert isinstance(data.get("keywords"), list) and len(data["keywords"]) <= 5
print(data)
```

## 4. 可测试：用 Fake LLM 把链先跑通

在没有模型 Key、或写单元测试时，用 Fake LLM 让链路可重复、可断言。

```python
from langchain_core.language_models.fake import FakeListLLM
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_messages(
    [("human", "把需求改写成验收标准：{req}")]
)
llm = FakeListLLM(responses=["Scenario: ok\n  Given ...\n  When ...\n  Then ..."])

chain = prompt | llm | StrOutputParser()
out = chain.invoke({"req": "新增 langchain 文档与示例"})
assert "Scenario" in out
```

这类测试通常用于验证：

- Prompt 变量是否齐全
- 组件拼装是否正确
- 输出解析是否可用

---

## 5. 工程实践建议（高频）

- **把 Prompt 当作配置**：避免散落在业务逻辑中，便于版本管理与回滚
- **为每条链定义输入/输出契约**：让调用方不需要“猜输出格式”
- **先离线跑通再接真实模型**：失败成本更低
- **不要在日志/Tracing 里泄露敏感信息**：密钥、PII、token 等
