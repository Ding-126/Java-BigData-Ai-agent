# Data Model: Flink 面试点整理（内容结构）

**Feature**: 001-flink-daydayup | **Phase**: 1

本需求为文档/内容型，无数据库；本文件描述“内容实体”及其关系与约束，用于指导文件组织与索引约定。

## 实体与属性

### 模块 (Module)

- **含义**: 对 Flink 面试点的分组，对应一个主题文件（如 `01-architecture.md`）。
- **属性**: 唯一英文短名（用于文件名）、中文标题、推荐顺序号、题目方向说明。
- **约束**: 至少 5 个核心模块（与 SC-002 一致）；每个模块对应一个 Markdown 文件。

### 面试点 / 知识点 (Topic)

- **含义**: Flink 某一知识领域（如 Checkpoint、状态后端、背压），可对应多道面试题。
- **属性**: 归属模块、可选难度标签（基础/进阶/生产）。
- **约束**: 跨模块知识点在主导模块下完整呈现，其他模块可引用（见 spec Edge Cases）。

### 面试题 (Question)

- **含义**: 单道问题，含题面与唯一标识或顺序号。
- **属性**: 题面文本、所属模块、可选难度、可选序号（如 Q1、Q2 或模块内编号）。
- **约束**: 每模块至少 3 题（SC-002）；题目须可独立理解、便于引用。

### 答案 (Answer)

- **含义**: 与面试题一一对应的详细解答。
- **属性**: 正文（可含小标题、列表、对比表）、适用版本（若涉及差异须标明，FR-005）。
- **约束**: 条理清晰、可独立理解；涉及专有概念时在答案内或同文件内说明（FR-002）。

### 文件 / 总览 (File / Index)

- **含义**: 承载“部分面试题+答案”的单个 Markdown 文件，或说明各文件主题与阅读顺序的索引（README）。
- **属性**: 文件名（约定见 contracts）、用途（模块内容 vs 总览）。
- **约束**: 总览必须说明划分方式与推荐阅读顺序（SC-004）；内容文件须通过总览或目录可发现（FR-004）。

## 关系

- **模块 1 : N 面试题**：一个模块对应多道面试题；每道题归属一个主导模块。
- **面试题 1 : 1 答案**：每题有且仅有一条详细答案。
- **模块 1 : 1 文件**：每个模块对应一个内容文件（如 `03-state-fault-tolerance.md`）。
- **总览 1 : N 模块**：README 引用并说明所有模块文件及推荐顺序。

## 状态与版本

- **内容版本**: 答案涉及 Flink 行为差异时须标明适用版本（如 1.17 vs 1.13）。
- **难度标注**: 题目或模块内可标注“基础/进阶/生产”，无状态机；仅用于查阅与选题。
