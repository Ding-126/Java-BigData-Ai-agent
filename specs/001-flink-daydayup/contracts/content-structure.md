# Contract: 内容结构与文件命名约定

**Feature**: 001-flink-daydayup | **Phase**: 1

本文档约定面试题与答案的文件组织方式及总览格式，供实现与查阅时一致遵循。

## 1. 文件命名

| 文件 | 用途 | 约定 |
|------|------|------|
| `README.md` | 总览/索引 | 必须存在；说明本目录用途、文件列表、各文件对应模块（中英文）、推荐阅读顺序、版本与难度标注说明。 |
| `01-architecture.md` | 架构与基础 | 模块内容文件；文件名格式为 `NN-<英文短名>.md`，NN 为两位数字顺序号。 |
| `02-api.md` | API 与编程模型 | 同上。 |
| `03-state-fault-tolerance.md` | 状态与容错 | 同上。 |
| `04-performance.md` | 性能与调优 | 同上。 |
| `05-production.md` | 生产与运维 | 同上。 |

- 模块内容文件仅包含“面试题 + 答案”，不包含规划类文档（plan、research、data-model、quickstart、contracts 等）。
- 新增模块时：新增 `NN-<英文短名>.md`，并更新 README 中的文件列表与推荐顺序。

## 2. 模块内容文件结构（每个 md 内）

- **标题**: 一级标题为模块中文名（如 `# 架构与基础`），便于目录与锚点。
- **题目与答案**: 每题使用二级标题为题面或题目标识（如 `## Q1 请简述 Flink 的整体架构`），紧跟该题的详细答案；可选用三级标题组织答案内的小节。
- **难度标注**: 在题目标题旁或题目前一行标明 `[基础]` / `[进阶]` / `[生产]`（可选但推荐），与 research 中“难度标注”一致。
- **版本说明**: 若某题答案仅适用于特定 Flink 版本，在该答案开头或末尾标明“适用版本：1.17”或“1.17 与 1.13 差异：…”。

## 3. 总览 README 必备章节

1. **本目录用途**：一句话说明（Flink 面试点整理、60k 岗位面试官视角）。
2. **文件列表**：列出所有模块文件，每行格式可为 `- [01-architecture.md](./01-architecture.md) — 架构与基础`。
3. **推荐阅读顺序**：按模块顺序 01→05，或另起一段说明“按基础→进阶→生产”的跨文件路径。
4. **版本与约定**：答案以 Flink 1.17 为主；难度标注含义（基础/进阶/生产）。

## 4. 索引与可发现性

- 查阅者通过 README 即可在 2 分钟内定位到任意模块对应文件（满足 SC-001）。
- 各模块文件内可使用目录（如 Markdown TOC）便于跳转到具体题目；不强制，以可读性为准。
