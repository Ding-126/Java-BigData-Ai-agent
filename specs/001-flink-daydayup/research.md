# Research: Flink 面试点整理（60k 岗位视角）

**Feature**: 001-flink-daydayup | **Phase**: 0

## 1. 文件组织方式

**Decision**: 按模块分文件 + 总览 README；模块内题目按“基础 → 进阶 → 生产”顺序排列。

**Rationale**:
- 规格要求“按模块或按易于理解、循序渐进”，按模块分文件最利于 2 分钟内定位（SC-001）。
- 同一文件内题目按难度/类型排序，兼顾“循序渐进”与“按模块精读”两种使用方式。
- 总览 README 满足 SC-004（5 分钟内理解整体结构）。

**Alternatives considered**:
- 仅按难度分文件（如 01-basics.md, 02-advanced.md）：不利于“只看状态与容错”的查阅路径，放弃。
- 单一大文件 + 锚点：单文件过大不利于维护与渐进阅读，放弃。

## 2. 模块与文件命名约定

**Decision**: 五类核心模块对应五个文件，英文短文件名便于工具与链接；README 中给出中文标题与推荐阅读顺序。

| 文件 | 模块（中文） | 建议题目方向 |
|------|--------------|--------------|
| `01-architecture.md` | 架构与基础 | 整体架构、部署模式、JobManager/TaskManager、任务链、Slot |
| `02-api.md` | API 与编程模型 | DataStream/DataSet、算子、窗口、时间语义、ProcessFunction |
| `03-state-fault-tolerance.md` | 状态与容错 | 状态后端、Checkpoint、Savepoint、Exactly-Once、状态恢复 |
| `04-performance.md` | 性能与调优 | 背压、反压、资源与并行度、状态与 RocksDB 调优、数据倾斜 |
| `05-production.md` | 生产与运维 | 监控、告警、升级与兼容、Flink SQL/Table、与 Kafka 等集成 |

**Rationale**: 与 spec 中 FR-001、SC-002 的 5 模块一致；英文文件名避免路径与工具兼容性问题。

**Alternatives considered**: 中文文件名（如 `01-架构与基础.md`）在部分环境下可能有编码或排序问题，故采用英文短名 + README 中文说明。

## 3. Flink 版本与答案适用范围

**Decision**: 答案以 Apache Flink 1.17 为主；若某题涉及 1.13/1.15 等版本行为差异，须在答案中单独标明“适用版本”或“1.17 与 1.13 差异”。

**Rationale**: FR-005 要求“涉及版本或配置相关差异须标明”；选定一个主版本便于表述一致，1.17 为当前常见 LTS 线。

**Alternatives considered**: 不固定主版本会导致答案表述分散；多版本并列会显著增加篇幅，故采用“主版本 + 差异处单独标注”。

## 4. 60k 岗位问题深度与数量

**Decision**: 每模块至少 3 题（满足 SC-002）；题目类型覆盖：基础概念（是什么）、原理与机制（为什么/怎么实现）、生产与选型（怎么用/怎么选）。难度在目录或题目标题旁标注（基础/进阶/生产），便于面试官选题与候选人按档位复习。

**Rationale**: 60k 档位预期为高级/专家岗，需同时考察概念、原理与实战；规格 Edge Cases 要求“标明难度/档位”。

**Alternatives considered**: 仅概念题或仅原理题会导致覆盖不全；不标难度会降低“针对性复习”的体验，故采用“类型覆盖 + 难度标注”。

## 5. 总览 README 必备内容

**Decision**: README 须包含：(1) 本目录用途（Flink 面试点整理、60k 岗位视角）；(2) 文件列表与各文件对应模块（中英文）；(3) 推荐阅读顺序（按模块顺序 01→05，或按“基础→进阶→生产”的跨文件路径）；(4) 版本与约定说明（答案以 1.17 为主、难度标注含义）。

**Rationale**: 满足 SC-004“总览说明划分方式及推荐阅读顺序”；满足 Constitution 便于查阅。

**Alternatives considered**: 仅列文件名不解释模块会延长“5 分钟内理解结构”的时间，故采用“列表 + 说明 + 推荐顺序”。
